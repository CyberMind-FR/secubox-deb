Eye Remote Dashboard Modes

How should the Eye Remote switch between different operating modes?

You mentioned several modes: metrics display (when SecuBox API available), icons mode (RPi Zero standalone), storage-only vs multi-gadget. Let's clarify the primary operating modes.

A

Two Modes: Connected vs Standalone

Connected: Real metrics from SecuBox API (OTG/WiFi)
Standalone: Local Pi Zero stats + icon menu for actions

Pros

  • Simple mental model
  • Auto-detect based on connectivity

Cons

  • Less granular control
B

Three Modes: Metrics / Icons / Storage

Metrics: Real SecuBox data when API available
Icons: Touch-free menu navigation (Pi Zero local)
Storage: USB mass storage only (boot media for ESPRESSObin)

Pros

  • Clear separation of concerns
  • Dedicated flash/recovery mode

Cons

  • Mode switching complexity
C

Four Modes: Swiss Army Full Control

Dashboard: Real-time metrics (API connected)
Local: Pi Zero self-monitoring + settings
Flash: USB storage + U-Boot console (recovery)
Gateway: Multi-SecuBox fleet aggregation

Pros

  • Maximum flexibility
  • Future-proof for fleet management

Cons

  • More UI states to design
  • Higher complexity
