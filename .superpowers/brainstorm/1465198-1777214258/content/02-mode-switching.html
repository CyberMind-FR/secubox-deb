Mode Switching Mechanism

You mentioned a flag file on storage to switch modes. How should this work?

Since touch is disabled (defective hardware), we need touchless control. Options for triggering mode changes:

A

Flag File on USB Storage

Create/delete files on the USB mass storage partition to trigger modes:
/SECUBOX_MODE_FLASH → Flash mode
/SECUBOX_MODE_LOCAL → Local mode
/SECUBOX_MODE_GATEWAY → Gateway mode
No file = Dashboard (default)

Pros

  • Works from any host (Windows/Mac/Linux)
  • No network needed
  • Survives reboots
B

Auto-Detect + Flag Override

Auto: Dashboard if SecuBox API responds, else Local
Override: Flag files force specific mode
/FORCE_FLASH → Always Flash mode
/FORCE_GATEWAY → Always Gateway mode

Pros

  • Smart defaults
  • Manual override when needed
  • Plug-and-play friendly
C

Config File (TOML/JSON)

Single config file on storage:
/SECUBOX.conf with:
mode = "auto" | "dashboard" | "local" | "flash" | "gateway"
Plus WiFi/Bluetooth settings in same file

Pros

  • All settings in one place
  • Structured, extensible
  • Can include WiFi/BT config
D

Hardware Button + Software Fallback

Physical button press cycles modes (GPIO on Pi Zero header)
+ Config file fallback for headless operation
+ LED blink patterns indicate current mode

Pros

  • Instant tactile feedback
  • Works without host connection

Cons

  • Requires hardware mod
