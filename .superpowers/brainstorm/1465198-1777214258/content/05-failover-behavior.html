Failover & Fallback Behavior

How should Eye Remote handle connection loss and mode transitions for "plug and play"?

Scenario: SecuBox API becomes unreachable

A

Immediate Fallback to Local

API timeout (3s) → instantly switch to Local mode
Show "Connection Lost" briefly, then Pi Zero stats
Auto-reconnect attempts every 10s in background

B

Graceful Degradation with Cache

Show last known metrics (grayed/stale indicator)
After 30s stale → transition to Local mode
Preserves context during brief disconnects

C

Hybrid: Cache + Visual Warning

Immediate: Show stale data with pulsing "OFFLINE" badge
After 15s: Fade rings to gray, keep time/hostname
After 60s: Full transition to Local mode
Best UX: User sees what's happening

Scenario: USB cable unplugged/replugged

1

Instant Re-detection

USB gadget re-enumerates automatically
Dashboard reconnects within 5s
No user action required

2

Show Connection Animation

Display "Connecting..." spinner during re-enumeration
Visual feedback that something is happening
Success: Green flash, Resume dashboard
