How ZKP Works

Zero-Knowledge Proofs allow one party (the prover) to prove to another party (the verifier) that they know a value, without revealing any information about the value itself.

SecuBox uses Hamiltonian path ZKP: proving knowledge of a path that visits every node in a graph exactly once, without revealing the path.

  1. Generate Key: Create a random graph with a known Hamiltonian path (the secret)
  2. Generate Proof: Create a cryptographic proof of knowledge without revealing the path
  3. Verify: Anyone can verify the proof is valid for the graph

Used in SecuBox for: mesh authentication, config validation, and secure parameter exchange.
